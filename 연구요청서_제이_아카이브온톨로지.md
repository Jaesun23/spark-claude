## 연구 목적

대량의 비정형 문서 아카이브를 온톨로지(객체 + 관계 그래프)로 변환하기 위한 **실전 사례, 기존 도구, 구현 패턴**을 조사한다. 이론보다 "실제로 누가 어떻게 했는가"에 초점을 맞춘다.

## 연구 범위

### 1. 문서 → 온톨로지 실전 사례 조사

실제로 대량 문서에서 지식 그래프/온톨로지를 구축한 사례:

- **금융/은행 분야**: 규정·업무매뉴얼을 온톨로지화한 사례 (RegTech, Compliance Knowledge Graph 등)
- **법률 분야**: 법률 문서에서 의무·권리·조건을 추출한 사례 (Legal Ontology, LKIF 등)
- **의료 분야**: 진료 가이드라인에서 의사결정트리를 추출한 사례 (Clinical Pathway)
- **기업 지식관리**: 사내 문서에서 지식 베이스를 구축한 사례

각 사례에서:
- 어떤 도구/기술을 사용했는가?
- 문서에서 객체와 관계를 어떻게 추출했는가?
- 결과물은 어떤 형태였는가?
- 성공/실패 요인은?

### 2. 팔란티어 온톨로지의 핵심 메커니즘

- Object Types, Link Types, Actions의 실제 모델링 예시
- "Ontology-first" 접근법: 왜 데이터보다 모델을 먼저 정의하는가?
- Palantir Foundry의 ontology editing workflow
- **아카이브 적용 시 차이점**: 팔란티어는 구조화된 데이터(DB, API)를 연결하지만, 우리는 비정형 문서에서 추출해야 함. 이 추출 단계에서 어떤 기술이 필요한가?

### 3. LLM 기반 지식 추출 기법

최근 LLM을 활용한 문서→온톨로지 변환 연구:

- LLM으로 문서에서 Named Entity + Relation을 추출하는 방법
- GraphRAG (Microsoft) 등 그래프 기반 RAG 접근법
- 프롬프트 설계: "이 문서에서 객체, 관계, 조건을 추출하라"를 어떻게 구조화하는가?
- 대량 문서에서 일관된 추출을 유지하는 방법 (스키마 고정, few-shot 예시 등)

### 4. 아카이브 유형별 패턴 비교

| 아카이브 유형 | 추출 대상 | 결과물 유형 |
|---|---|---|
| 업무규정/방법서 | 절차, 조건, 전결권, 예외 | 의사결정트리, 업무 위저드 |
| 연구 논문 | 가설, 방법, 결과, 인용 | 지식 맵, 연구 갭 분석 |
| 기술 문서 | 컴포넌트, API, 의존성 | 아키텍처 다이어그램, 영향 분석 |
| 법률/규정 | 주체, 의무, 조건, 제재 | 컴플라이언스 체크리스트 |

**핵심 질문**: 유형이 달라도 추출 프로세스의 공통 패턴이 있는가?

### 5. 저장 및 활용 도구

- 지식 그래프 저장: Neo4j, Redis Graph, 노션 DB로 가능한 범위
- 시각화: 온톨로지를 시각적으로 탐색하는 도구
- 쿼리: 온톨로지에서 "담보 변경 시 필요한 절차는?"을 검색하는 방법
