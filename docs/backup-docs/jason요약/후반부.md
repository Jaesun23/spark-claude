## 작업분해 (Task Breakdown)
  1. 독립 실행 및 테스트 가능한 완전한 레고블럭으로 나누기(TODO, pass 없이)

- "레고블럭" = "단위작업" = 의미있는 기능이 있고, 단독으로 테스트 가능하며, 명확한 입력/출력이 있는가?

  2. 두 가지 완성 조건

- 일관성 (Consistency):

  - 모든 Task가 같은 도구 사용 (DNA 시스템 환경 기반)

  - 모든 Task가 같은 패턴 따름

  - 모든 Task가 같은 표준 준수

- 누락없음 (No Omissions):

  - Blueprint의 모든 기능이 Task로 분해됨

  - Task 목록 = Blueprint 기능 목록

  - 빠진 Task 없음

  3. God Object 회피 원칙

- 단일 파일: 500줄 이하 (가독성, 유지보수 기본 원칙)
- Task 크기: 400~800줄 (구현 + 테스트)
  → 80K 실제 작업 공간 기준
  → 간단 200~400줄 / 표준 400~600줄 / 복잡 600~800줄
- 800줄 초과 예상: 무조건 모듈화 (수평 분할)

  4. 컨텍스트 제약 이해

- 200K 총 컨텍스트
  → Write 작업: 2배 소모 (생각 + 쓰기) = 100K 가용
  → 오버헤드 (시스템, 도구, 명령어): 20K
  → 실제 작업 공간: 80K
- Agent 하나당:
    ├─ 체크리스트 (5K)
    ├─ DNA 시스템 참조 (15K)
    ├─ 관련 인터페이스 (10K)
    ├─ 도메인 모델 (10K)
    ├─ 구현 공간 (30K)
    └─ 테스트 코드 (10K)

  5. Task 분할 방식

- Case 1: 모듈화 분할(수평분할)
  - 재사용 가능한 독립 모듈들을 먼저 만들고, 나중에 조립
  - 재사용성↑ or 복잡도↑ → 사전 모듈화 필수
  - 500줄 초과 예상 → 무조건 모듈화
- Case 2: 단계별 구현(수직구현)
  - 전체 기능을 하나씩 단계별로 완성
  - 간단한 구조, 빠른 결과 확인

  6. Agent 병렬 작업 최적화

- 2호 단독: 80K (조율 + 구현 병행, 빡빡)
- 2호 + Agent N개: 80K × (1+N)
  → 2호: Blueprint 관리 + 조율만 (여유↑)
  → Agent들: 각자 독립 Task 구현 (병렬)
  → 독립 파일 = Git 충돌 없음 = 병렬 가능

## 체크리스트 (Checklist)
  1. 작업분해 단계에서 분해한 Task별 작업 목록
  
  2. TDD 필수 원칙
- 구현 후 테스트 = 구현이 정답 (의도 이탈, 불필요한 기능 추가)
- TDD = 의도가 정답 (원래 계획 준수, 필요한 기능만 구현)
- AI는 과도한 일반화 경향 → 테스트로 범위 제한
- 테스트 = 실행 가능한 명세 = Blueprint 의도 구현 보장

  3. TDD기반 9-Step Checklist만으로 모든 기능 구현, 0 violations, 95%+ coverage 보장
  
  4. 체크리스트 완성 조건: 실행가능성, 자급자족성, 검증가능성
  
  5. 체크리스트 최적화 (5K 목표)
- Task 목표 (1줄)
- 입출력 명세 (10줄)
- 9-Step 세부 지침 (각 20줄 = 180줄)
- 검증 기준 (10줄)
- 자급자족(self-contained)하되, 참조는 최소화

  6. 실제 작업을 실행할 2호/Agent를 위한 완전한 정보

## 구현 (Implementation)

  1. 체크리스트만 읽고 실행 + Project Standards 준수 + 자동화 강제
  2. 전체 컨텍스트 불필요!

- Agent는 현재 Task 체크리스트만 참조
- DNA 시스템 인터페이스만 참조
- 의존 Task는 API/인터페이스만 참조

  3. 병렬 실행

- 각 Agent가 독립 파일 작업
- Git 충돌 없음
- 자동 통합 가능

  4. 블럭 하나씩 완성해 나가면 전체가 완성!
