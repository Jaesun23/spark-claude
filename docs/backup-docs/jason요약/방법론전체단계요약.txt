Stage 1: 패밀리 구분과 핵심기능 파악
 - 3-Layer Decision Tree에 따른 패밀리 구분
   ➟ 지금은 5개로 나누고 있는데 논리적 오류가 있음. 
     경우의 수는 2x3x3 =18 18가지인데 현재 5가지 밖에 없음.
     서로 상충되어 실현 불가능 패밀리가 있다고 해도 실제 패밀리는 7~10가지 정도 예상됨
     이에 대해 1호가 연구리서치 중
 - 발견한 문제 또는 실현하고자 하는 것의 가장 "핵심기능"이 무엇인지 파악
Stage 2: 구조설계
 - 패밀리와 핵심기능 파악을 통해 그 시스템에서 결정해야 할 요소들의 목록을 만듭니다.
 - 더 명확한 결정을 위해 가용한 내부자원과 제약조건들을 파악하고, 더불어 외부의 제약과 기회들을 파악 
Stage 3: 결정 문서화 (ADR)
 - 파악된 결정 요소들에 대해 하나씩 ADR을 작성 ➟ 너무나도 당연하다고 생각되는 요소들도 모두 작성
 - 각 요소들에 대해서는 가용자원, 내/외부 제약조건을 고려한 가장 최선의 방안으로 결정
 - Bootstrap 요소들과 도메인별 요소들을 구분해서 ADR을 작성
Stage 4: Bootstrap 계획
 - 전 프로젝트에 공통으로 사용되고, 지켜져야 할 '환경'구축을 계획
 - common/ 에 들어갈 모듈들에 대한 ADR을 바탕으로 전체 DNA 기본시스템의 청사진 작성
 - Bootstrap 청사진을 기초로 작업분해 및 개별 체크리스트 문서 작성
Stage 5: Bootstrap 실행
 - Bootstrap 체크리스트에 기반하여 DNA 기본시스템 구현
 - 구현된 모듈들의 작동 점검
Stage 6: Project Standards 작성
 - 프로젝트 종료 시까지 지켜야 할 코딩 스타일, 로깅, 에러, 설정, 기타 룰에 대한 문서 작성
 - Bootstrap 단계에서 구현된 공통모듈 사용 강제조항 포함
Stage 7: Project Blueprint 작성
 - 초 상세한 프로젝트 전체에 대한 청사진 문서 작성
 - 아키텍처 구조, 도메인 구조, 전체 시스템/도메인별 다이어그램
 - 도메인별 구현해야 할 것들을 작성
 - 모든 구현은 최신의 기술과 기법으로 구현이 되도록 작성
Stage 8: Task Breakdown 문서 작성
 - AI가 집중해서 완전하게 구현할 정도의 크기로 작업을 분할
 - 작업분할 기준에 따른 작업분할
Stage 9: 작업별 Checklist 작성
 - 분할된 작업별로 체크리스트 문서 작성
 - TDD기반 9Step-Checklist
 - 1:목표이해 → 2:테스트작성 → 3:구현 → 4:정적검증 → 5:단위테스트실행 → 6:리팩토링 → 7:종합테스트 → 8:문서화 → 9:커밋
 - 최종목표: 모든 기능 구현, 검증결과 0 violations, 테스트 95%+ coverage
