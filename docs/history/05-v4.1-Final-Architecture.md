### Layer 2: Quality Gates (spark_quality_gates.py)
**Role**: Quality validation and enforcement

```python
def enforce_quality_gates(state_json):
    """
    Mandatory quality validation.
    Returns: PASSED or FAILED with details
    """
    # Check violations
    total_violations = state_json["quality"]["violations_total"]

    if total_violations > 0:
        return {
            "status": "FAILED",
            "message": f"Fix {total_violations} violations",
            "can_proceed": False
        }

    return {
        "status": "PASSED",
        "message": "All quality gates satisfied",
        "can_proceed": True
    }
```

## 🎯 Token Safety Protocol

### Constraints
- **Hard limit**: 200K tokens per request
- **Practical limit**: 90K tokens (safety margin)
- **Agent sizes**: ~1K (team) to ~3.9K (implementer-spark)

### Note on Write Operations
**Important**: Write operations = 2x token consumption
- Saving to memory (1x)
- Generating output (1x)
- Total: 2x token consumption

### Token Budget Management
```
Total: 200K tokens
Safety: 90K tokens (45%)

Allocation:
- Orchestrator (2호): ~5K tokens
- Agents (2-3): ~8-12K tokens
- User context: Variable
- Safety buffer: ~70K tokens
```

## 🔄 Transition to Next Version

### Why Move to v4.2

v4.1 was complete, but room for improvement existed:

1. **Trait-based Persona introduction**
   - Further systematize agent definitions
   - Separate Traits + Protocols
   - Add code-based behavior

2. **Dual-layer Enforcement**
   - Persona (document) + Code (script)
   - Dual verification system
   - Stronger quality assurance

3. **Plugin preparation**
   - Distributable structure
   - `~/.claude/` or `project/.claude/`
   - Improved reusability

**v4.2 Goal**: Trait-based system + Plugin architecture
