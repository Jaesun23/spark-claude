# v5.0 — Standard-Based Refactoring (2026-03-28)

## The Core of This Version

SPARK evolved from a "plugin" into an "agent definition standard."

As Anthropic's 6 persona research papers scientifically validated the principles that SPARK/TRAITS had discovered empirically, the project was fully restructured around `AGENT_DEFINITION_STANDARD.md` — integrating three sources: SPARK practice + TRAITS theory + Anthropic science.

## Key Changes

### Project Character Shift
- Before: SPARK plugin (22 agents + 12 commands + infrastructure)
- After: Agent definition standard + 3 reference agents for software development

### Agent Reselection
- 22 agents (7 core + 15 team) → 3 agents (implementer, diagnostician, auditor)
- Selected based on independent personality clusters:
  - **implementer** — builds (implementation + testing + quality)
  - **diagnostician** — investigates and illuminates (analysis + diagnosis + debugging) — merged from existing analyzer + diagnostician
  - **auditor** — validates against standards (audit + review + judgment) — entirely new
- `-spark` suffix removed — clean naming as universal reference agents

### Core Design Shift
- Python Behavior Protocol code blocks **completely removed**
  - Rationale: PSM (Persona Selection Model) — code-form rules activate the "rule executor" personality cluster
  - Rationale: Character Training — narrow rules trigger unintended personality inferences
- External enforcement framing → **personality expression framing**
  - "Must not make claims without file paths" → "Claims without evidence feel incomplete"
- AGENT_DEFINITION_STANDARD.md 7 core principles applied

### Structural Changes
- `spark-plugin/` → `archive/v5.0-standard-refactoring/spark-plugin/`
- Methodology guides, commands, and team agents all archived
- New `agents/` directory with 3 reference agents
- 2 research documents added (TRAITS summary, Anthropic persona research summary)
