## 📋 Background

In v1.0, we discovered the token inefficiency problem in SuperClaude (44K tokens). Now a theoretical foundation was needed to solve it. Rather than simply "cutting things down," a systematic transition to a Multi-Agent System was required.

## 🔬 Research Topics

### 1. Multi-Agent System (MAS) Formalization

#### Redefining Personas as Agents
```
Before (SuperClaude):              After (SPARK concept):
┌──────────────┐                  ┌──────────────┐
│   Persona    │                  │    Agent     │
│  (Static)    │      →           │  (Dynamic)   │
│  Always On   │                  │  On-demand   │
└──────────────┘                  └──────────────┘
```

#### Agent Characteristics
- **Autonomy**: Operates independently
- **Isolation**: Cannot directly call other agents
- **Tool-based**: Activated only via Claude Code's Task tool
- **Stateless**: State managed via JSON

### 2. Persona Definition Language (PDL)

A meta-model developed to standardize agent structure:

```markdown
---
name: {agent-name}
role: {primary-responsibility}
expertise: {specialized-domain}
triggers: {when-to-activate}
---

# Traits
- trait1
- trait2

# Protocol
## Phase 1: ...
## Phase 2: ...
```

### 3. Dynamic Weight System Formalization

#### 2-Phase Selection Process
1. **Phase 1**: Task analysis → Identify required agents
2. **Phase 2**: Calculate agent weights → Allocate token budget

#### Weight = Token Budget
```
Agent Weight    Token Allocation    Usage
0.0            0 tokens            Not loaded
0.3            ~1,500 tokens       Minor role
0.7            ~3,500 tokens       Major role
1.0            ~5,000 tokens       Primary role
```

### 4. Conflict Resolution Hierarchy

Priority when multiple agents are needed:
```
Security > Performance > Functionality
```

Example: Security requirements take priority over performance optimization

## 🎯 Birth of the SPARK Concept

### SPARK Acronym
**SPARK = Subagent Performance Architecture with Reduced toKens**

### Core Design Principles

1. **Each agent as independent tool**
   - Agent activated via Claude Code's Task tool
   - Direct agent-to-agent communication forbidden

2. **On-demand loading**
   - No pre-loading
   - Loaded only when needed

3. **Task-based activation**
   - Orchestrator (2호) analyzes the task
   - Selects and activates the appropriate agent

4. **Token efficiency**
   - Average of only 2-3 agents loaded
   - Goal: 44K → 5K tokens (88% reduction)
