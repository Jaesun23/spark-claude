4. Blueprint AI

- 앞선 프로젝트의 실패에서 AI와의 협업에 반드시 필요한 요소에 대해 본격적으로 고민해보기 시작함.
- 가장 중요한 것은 프로젝트의 시작에서 종료까지 전체 계획을 제대로 세우고, 계획에 따라서 진행하되 각 단계가 하나의 흐름으로 이어질 수 있도록 연결해야 한다는 걸 깨닫게 됨. 그 수단을 "문서"로 하기로 결정함.
- 모든 작업은 문서로 계획되고, 결과도 문서로 정리되어 문서를 통한 컨텍스트 관리와 프로젝트 진행 관리가 되는 프레임워크를 구상하고 진행함.
- 이 때 앤트로픽에서 MCP를 발표했고, 그걸 이용하고자 계획을 세우고 있었는데 Claude는 이걸 CLI 명령어 시스템으로 이해하고 전혀 다른 계획을 세움.
- 모든 계획을 MCP에 맞게 다시 짜고 문서화 하기 시작했는데 진행하던 도중 전체 소프트웨어 개발 프로세스에 대한 이해가 부족하다는 것과 각 단계별 자동화 프로세스가 명확하지 않아 계획 문서 수정 중 중단. 당장 모든 단계에서 쓰일 문서의 기본양식들을 정해야 함에도 그 내용을 무엇으로 해야 할 지 경험도 지식도 없었음.
- 조금 더 소프트웨어 개발 프로세스와 각 단계의 작업 검증과 연결의 자동화 방법을 연구한 후에 진행하기로 함.
