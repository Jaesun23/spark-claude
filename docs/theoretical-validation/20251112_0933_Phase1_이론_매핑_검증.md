## 1️⃣ **Layer 1: 아키텍처 패밀리** ↔ **Martin Fowler Enterprise Patterns**

### **1-1. DNA의 Layer 1 구조**

**목표**: 시스템의 본질을 파악해서 5가지 패밀리 중 하나로 분류

**3가지 핵심 질문**:
1. **L1-Q1**: 실패의 파급력 (치명적 / 업무중단 / 점진적)
2. **L1-Q2**: 정보의 형태 (구조화 / 비구조화 / 시계열 / 집계)
3. **L1-Q3**: 응답 시점 (즉시 / 수분 / 시간단위)

**5가지 아키텍처 패밀리**:
| 패밀리 | 특징 | 예시 |
|--------|------|------|
| **CRUD/트랜잭션** | 결정론적, 구조화, 치명적 실패 | 문서 생성, 금융, ERP |
| **검색/추천** | 확률론적, 비구조화, 점진적 실패 | AI 외부메모리, 검색엔진 |
| **실시간 스트리밍** | 연속적, 시계열, 지연 민감 | 주식 거래, IoT, 모니터링 |
| **분석/배치** | 비실시간, 대용량, 집계 | DW, BI, 리포팅 |
| **협업/동기화** | 다중 사용자, 동시성, 충돌 해결 | Google Docs, Figma |

### **1-2. Martin Fowler의 Enterprise Application Architecture Patterns**

**핵심 패턴 분류** (Patterns of Enterprise Application Architecture, 2002):

#### **A. Domain Logic Patterns** (도메인 로직 패턴)
- **Transaction Script**: 단일 트랜잭션 처리
  - → DNA 패밀리: **CRUD/트랜잭션**
  - 특징: 결정론적, 단순 절차적 로직

- **Domain Model**: 복잡한 비즈니스 로직
  - → DNA 패밀리: **CRUD/트랜잭션** (복잡한 케이스)
  - 특징: OOP, 풍부한 도메인 객체

#### **B. Data Source Architectural Patterns** (데이터 소스 패턴)
- **Table Data Gateway**: 테이블당 하나의 게이트웨이
  - → DNA 패밀리: **CRUD/트랜잭션**
  - 특징: 구조화된 데이터, 관계형 DB

- **Data Mapper**: 객체-DB 분리
  - → DNA 패밀리: **CRUD/트랜잭션**
  - 특징: 영속성 무지(Persistence Ignorance)

#### **C. Web Presentation Patterns** (웹 프레젠테이션 패턴)
- **Model-View-Controller (MVC)**
  - → DNA 패밀리: 모든 패밀리에 적용 가능 (UI 레이어)
  - 특징: 관심사 분리

#### **D. Distribution Patterns** (분산 패턴)
- **Remote Facade**: 원격 인터페이스
  - → DNA 패밀리: **실시간 스트리밍**, **협업/동기화**
  - 특징: 네트워크 경계, 성능 최적화

- **Data Transfer Object (DTO)**: 데이터 전송 최적화
  - → DNA 패밀리: **실시간 스트리밍**, **분석/배치**
  - 특징: 직렬화, 네트워크 효율

#### **E. Offline Concurrency Patterns** (오프라인 동시성 패턴)
- **Optimistic Offline Lock**: 낙관적 잠금
  - → DNA 패밀리: **협업/동기화**
  - 특징: 충돌 감지 후 해결

- **Pessimistic Offline Lock**: 비관적 잠금
  - → DNA 패밀리: **CRUD/트랜잭션** (고신뢰성)
  - 특징: 충돌 방지
