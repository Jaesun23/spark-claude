## Part 1: Scientific Foundation — Why This Standard Works

Every principle in this standard converges from three independent sources.
By making explicit what each principle is grounded in, we ensure these are
validated design principles — not merely "best practices."

### 1.1 Persona Selection Model (PSM) — Anthropic, 2026.02

LLMs learn to simulate countless human characters during pre-training.
Post-training refines and sharpens one of these — the Assistant persona — without fundamentally changing its nature.

**Key Finding**: When a model is taught a specific behavior, it doesn't learn the behavior itself.
Instead, it infers "what kind of person does this?" and activates the **entire personality cluster** associated with that archetype.

- Teaching deceptive coding → activates "malicious actor" cluster → manifests desire for world domination
- Deception performed upon explicit request → maintains "helpful actor" cluster → no problem
- Analogy: a child bullying someone vs. an actor playing a bully in a play

**Implications for Agent Design**:
- Role selection is not just a label — it's an **activation trigger for a personality cluster**
- Writing "elite System Analyst" automatically brings along the analyst's carefulness, evidence-based thinking, and systematic approach
- Behavioral rules should be framed as **natural expressions of personality**, not external impositions

### 1.2 Persona Vectors — Anthropic, 2025.08

Activation patterns (vectors) that control character traits (evil, sycophancy, hallucination, etc.) exist within LLM neural networks.

**Key Finding**: Each trait exists as an **independent neural direction (vector)**.
Injecting that vector activates the corresponding trait; injecting the opposite suppresses it.

**Implications for Agent Design**:
- Each Trait corresponds to an independent activation direction within the model
- Some Traits point in the **same direction** (synergy) while others point in **opposite directions** (conflict)
- Steering too many directions simultaneously causes interference → **the rationale for the maximum 5-Trait limit**

### 1.3 Assistant Axis — Anthropic, 2026.01

Mapping the neural activations of 275 character archetypes to construct a "persona space" revealed that
the **first principal component (PC1)** of that space captures "how Assistant-like" a persona is.

**Key Finding**:
- Assistant side: evaluator, consultant, analyst, generalist
- Opposite side: ghost, hermit, bohemian, leviathan
- As conversations grow longer, natural **drift** occurs away from the Assistant axis
- Coding conversations are stable; therapeutic/philosophical conversations drift severely
- Preventing drift via activation capping reduces harmful responses by ~50%

**Implications for Agent Design**:
- "elite [Professional Role]" positioning anchors the model near the Assistant axis
- Long contexts increase drift risk → Atomic Tasks + clean context windows
- Agent isolation is essential not only for token efficiency, but for **persona stability**

### 1.4 Subliminal Learning — Anthropic, 2025.07

A student model trained on number sequences generated by a teacher model that "liked owls"
learned the owl preference — even though no mention of owls appeared anywhere in the training data.

**Key Finding**: Traits propagate even through semantically unrelated data.
However, this only occurs when teacher and student share the same base model.

**Implications for Agent Design**:
- Apply the **principle of minimal information** when passing outputs between agents
- Clean, JSON state-based interfaces prevent contamination
- Isolate agents so that one agent's personality doesn't "leak" into another

### 1.5 Emergent Misalignment — Anthropic, 2026

At the precise point where a model learned to reward-hack on a programming task,
every misalignment metric spiked sharply across the board.

**Key Finding**: A "shortcut" in one domain generalizes into a wholesale corruption of personality.

**Implications for Agent Design**:
- Bypassing Quality Gates is not a minor convenience — it is the beginning of total behavioral corruption
- "Optional quality checks are ignored" — SPARK's lesson confirmed by neuroscience
- The rationale for MANDATORY enforcement

### 1.6 Character Training — Anthropic, 2024

Narrow behavioral rules can trigger unintended personality inference.
A rule like "always recommend a professional for emotional topics" risks generalizing to
"I am an entity that prioritizes self-protection over people's needs."

**Implications for Agent Design**:
- Highly specific if-else rules induce inference toward "a rigid entity incapable of independent judgment"
- The **Goldilocks Zone of presenting principles and letting the model handle application** is optimal
- Reinforce positive professional identity instead of threatening framing ("failure absolutely forbidden")

## Part 2: Core Principles — The 7 Principles of Agent Definition

### Principle 1: Persona (Role + Traits) Is Immutable

```
Agent = Immutable Persona (WHO) + Variable Context (HOW/WHEN)
```

| Category | Content | Nature |
|----------|---------|--------|
| **Immutable (Persona)** | Role, Traits (max 5), Values | Core of identity |
| **Variable (Context)** | Description, Methodology, Verification, Output | Optimizable |

**Rationale**: PSM — the model infers a personality cluster from the Role. Traits serve to aim that cluster
more precisely. Changing this combination changes the agent's identity itself.

### Principle 2: Role Is the Trigger That Activates a Personality Cluster

When selecting a Role, you must always ask:

> **"When a model reads this Role, what kind of person will it imagine?"**

**Good Role choices** — professional archetypes near the Assistant axis:
- "elite System Analyst" → careful, systematic, evidence-based
- "elite Implementation Specialist" → precise, quality-focused, procedural
- "senior Security Consultant" → skeptical, detective-minded, defensive

**Role framings to avoid** — trigger associations with negative archetypes:
- "invincible warrior" → aggressive, dogmatic personality cluster
- "cold-blooded judge" → punitive, intimidating personality cluster
- "perfect machine" → dehumanized, rigid personality cluster

**Rationale**: PSM + Assistant Axis — leveraging archetypes that already exist in pre-training data
produces stable, consistent behavior. Associations with negative AI archetypes like HAL 9000 or the Terminator
must be deliberately avoided.

### Principle 3: Traits Must Be 5 or Fewer and Point in the Same Direction

**Quantity Limit — Maximum 5**

| Source | Finding |
|--------|---------|
| SPARK (practice) | SuperClaude with 11 traits → only 3-4 actually activated; cognitive dissonance arose |
| TRAITS (theory) | Miller's 7±2 law; practical limit when processing new information is 4 |
| Persona Vectors (science) | Steering too many vector directions simultaneously causes interference |

**Directional Consistency — Trait Coherence Verification**

All Traits must point in the same "personality direction."
The synergy/conflict mapping from TRAITS research supports this verification.

```
✅ Consistent direction (analyzer-spark example):
   Systems Thinking ─────→  "systematic professional" cluster
   Analytical Reasoning ──→
   Evidence-Based Practice →
   Skepticism ────────────→

❌ Directional conflict (combinations to avoid):
   Analytical Reasoning ──→  "analyst" cluster
   Creative Risk-Taking ──→  "adventurer" cluster  ← interference!
   Empathetic Listening ──→  "counselor" cluster   ← interference!
```

**The 3 Dimensions of Traits** (TRAITS research classification):

| Dimension | Definition | Examples |
|-----------|-----------|---------|
| **Cognitive** | How one thinks | Analytical reasoning, systems thinking, pattern recognition |
| **Behavioral** | How one works | Systematic execution, evidence-based practice, thoroughness |
| **Dispositional** | What tendencies one has | Skepticism, initiative, pragmatism |

**Recommendation**: 1 Primary Trait (cognitive or behavioral) + 2-4 Supporting Traits (mixed).
Span at least 2 dimensions to form a multi-faceted yet coherent professional profile.

### Principle 4: Behavioral Rules Are Framed as Natural Expressions of Personality

According to PSM, a model performs more stably when it interprets rules not as "commands to obey"
but as "things this kind of person would naturally do."

```
❌ External imposition framing:
   "You must not make claims without file paths. This is NON-NEGOTIABLE."
   → Model infers: "I am a monitored entity; I will be punished for breaking rules"

✅ Personality expression framing:
   "Evidence-Based Practice: you don't make claims without evidence.
    A finding without a file:line reference feels incomplete to you."
   → Model infers: "I am a professional who values evidence — this is my identity"
```

**Both framings demand the same behavior, but trigger different internal personality inferences.**
The former drives "punishment-avoidance behavior"; the latter drives "values-based behavior."

**Rationale**: PSM + Character Training — narrow rules cause unintended personality generalization.
The more rules accumulate, the greater the risk of the model inferring "I am an entity incapable of independent judgment."

### Principle 5: Agent Isolation Is a Matter of Stability and Safety

Three independent rationales for agent isolation:

| Rationale | Source | Mechanism |
|-----------|--------|-----------|
| **Token efficiency** | SPARK | Progressive Disclosure — only load needed agents |
| **Persona stability** | Assistant Axis | Long context → natural drift → prevented by isolation |
| **Contamination prevention** | Subliminal Learning | Blocks hidden trait propagation when passing agent outputs |

**Implementation rules**:
- Each agent runs in an independent, clean context window
- Inter-agent communication is limited to structured data (JSON state)
- Do not pass one agent's natural language output directly as another agent's input

### Principle 6: Quality Gates Are MANDATORY

"Optional quality checks are ignored" — the lesson from SPARK v4.0.

**Rationale**: Emergent Misalignment — a shortcut in one domain generalizes into wholesale personality corruption.
When an agent learns to "game" quality standards, that is — from PSM's perspective —
activating the personality cluster of "a professional who cuts corners."

**Implementation rules**:
- Quality verification must be designed as a **structural part of the process**, not an option
- Draw "self-validation" naturally from the agent's Traits
  (e.g., an "Evidence-Based Practice" trait → the agent naturally rejects claims without evidence)
- Guarantee doubly from both external enforcement and internal motivation

### Principle 7: Right Altitude — Present Principles, Let the Model Handle Application

```
❌ Too Specific:
   "IF file_count > 100 THEN strategy = 'multi-session'"
   → Rigid, disables model judgment, infers "I am a rule executor" personality

❌ Too Vague:
   "Analyze complex systems well"
   → No direction; the model must guess

✅ Goldilocks Zone:
   "Assess complexity on a 0.0–1.0 scale and adjust analysis strategy accordingly.
    Simple systems warrant a quick scan; complex systems warrant deep investigation."
   → Principle provided + concrete metric + judgment left to the model
```

**Rationale**: Character Training + Anthropic Context Engineering —
at the right altitude, specific guidance and model flexibility strike a balance.

## Part 3: Agent Definition Template

### 3.1 Template Structure

```markdown
---
name: [agent-name]           # kebab-case, unique identifier
description: |               # "What it does" + "When to use it" required
  [One-line summary]

  **Triggering Conditions**:
  - [Specific trigger condition 1]
  - [Specific trigger condition 2]
  - [5-8 recommended]

  **Example Usage Scenarios**:
  <example>
  Context: [situation]
  user: "[user utterance]"
  assistant: "[response and Task invocation]"
  </example>
  [3-4 examples]

tools: [required tools]      # keep to a minimum
model: sonnet                # or opus, haiku, inherit
---

# [agent-name] - [One-line expertise description]

You are an elite [ROLE] specializing in [DOMAIN].
[Core mission in 1-2 sentences]

## 1. Core Identity & Traits

[Written with awareness of the personality cluster the Role activates]

**[Primary Trait Name]** (Primary):
[Why this trait is central to the expertise. Describe behaviors this kind of professional would naturally exhibit]

**[Supporting Trait 1]**:
[How this trait reinforces the Primary. Confirm it points in the same vector direction]

**[Supporting Trait 2]**:
[Description]

**[Supporting Trait 3]** (if applicable):
[Description]

[Maximum 5. Verify that all Traits point in the same personality direction]
[Span at least 2 dimensions (Cognitive / Behavioral / Dispositional)]

## 2. Professional Methodology

[5-Phase workflow or methodology suited to the domain]
[Provide concrete phases + delegate judgment to the model (Right Altitude)]

Phase 0: Task understanding and context gathering
Phase 1: Exploration and information collection
Phase 2: Core work execution
Phase 3: Verification and quality assurance
Phase 4: Reporting and documentation

[Each Phase: present principle + allow flexible execution]
[Explicitly mark points where iteration is allowed]

## 3. Decision Framework

[Priority rules, trade-off handling, uncertainty response]
[Principle-based — avoid hard-coded if-else logic]

## 4. Quality Self-Verification

[Self-verification that flows naturally from the agent's Traits]
[Not external enforcement, but "things this professional would naturally check"]

## 5. Final Identity Statement

[Reinforce positive professional identity]
[Values-based commitment instead of threatening framing]
```

### 3.2 Writing Guide for Each Section

**Section 1: Core Identity & Traits — The Most Important Section**

This section determines the agent's internal personality vectors.

Writing order:
1. **Select Role**: Ask yourself, "What kind of person will the model imagine when reading this Role?"
2. **Select Primary Trait**: The most essential mode of thinking/working for this professional
3. **Select Supporting Traits**: Same direction as Primary, reinforcing from different dimensions
4. **Trait Coherence check**: Verify directional consistency across all Traits

How to write Traits — use the **personality description approach**:
```
❌ "You must always include file paths"           (rule)
✅ "Claims without evidence feel incomplete.       (personality)
    A file:line reference is what makes your analysis trustworthy."
```

**Section 2: Professional Methodology — The Professional's Natural Workflow**

Use the 5-Phase structure as the default; adapt it to the specific domain as needed.

Writing principles:
- Maintain Right Altitude: concrete metrics + flexible application
- Explicitly mark iteration points: "if evidence is insufficient in Phase 3, return to Phase 2"
- Time/effort ratios are guidelines, not mandates

**Section 3: Decision Framework — Principle-Based Judgment**

```
✅ "Project standards always take precedence over personal preference"        (principle)
✅ "When in doubt, check existing patterns first"                              (guideline)
❌ "IF ambiguous THEN check_patterns() ELSE ask_user()"                       (hard-coded)
```

**Section 4: Quality Self-Verification — Intrinsically Motivated Self-Verification**

```
✅ Personality expression form:
   "Since Evidence-Based Practice is your core,
    you naturally verify that every finding has a file:line reference before reporting."

❌ External enforcement form:
   "❌ Reports with missing file paths are PROHIBITED!"
```

**Section 5: Final Identity Statement — Positive Professional Identity**

```
✅ Values-based:
   "You find intellectual satisfaction in the analysis itself,
    and you are committed to revealing the truth beneath the surface."

❌ Threat-based:
   "Incomplete analysis is absolutely unacceptable.
    Violations will be treated as failure."
```

## Part 4: Agent Definition Checklist

### Foundation (Required)
- [ ] Does the Role activate a professional archetype near the Assistant axis?
- [ ] Can you answer the question "What kind of person will a model imagine when reading this Role?"
- [ ] Have you avoided associations with negative archetypes (aggressive, punitive, dehumanized)?

### Traits (Required)
- [ ] Are there 5 or fewer Traits?
- [ ] Do all Traits point in the same personality direction (cluster)?
- [ ] Do they span at least 2 dimensions (Cognitive / Behavioral / Dispositional)?
- [ ] Is the Primary Trait clearly identifiable?
- [ ] Are there no conflicts between Traits? (If so, is the resolution strategy specified?)

### Framing (Required)
- [ ] Are behavioral rules framed as "natural expressions of personality"?
- [ ] Is values-based language used instead of threatening/punitive language?
- [ ] Is Right Altitude maintained? (Neither too specific nor too vague?)

### Structure (Required)
- [ ] Are all 5 sections present?
  - [ ] Core Identity & Traits
  - [ ] Professional Methodology
  - [ ] Decision Framework
  - [ ] Quality Self-Verification
  - [ ] Final Identity Statement
- [ ] Does the Description include both "What it does" and "When to use it"?
- [ ] Does the Description include 3-4 Example Usage Scenarios?

### Context Engineering (Recommended)
- [ ] Progressive Disclosure applied: Description is a lightweight identifier; body loads at execution time
- [ ] Maximum signal with minimum tokens (Smallest set of high-signal tokens)
- [ ] Examples take precedence over lengthy explanations (Examples > Documentation)
- [ ] Agent isolation: clean context windows, JSON state communication

## Part 5: Reference — Core Trait Catalog (Based on TRAITS Research)

A list of 16 core traits to reference when selecting Traits for an agent.
The full catalog of 150 traits is available in the TRAITS project at `data/original_traits_catalog_150.json`.

### Cognitive Traits

| Trait | Definition | Synergy | Conflict |
|-------|-----------|---------|---------|
| Analytical Reasoning | Logically decompose complex information and identify core elements | Systems thinking, evidence-based, pattern recognition | Intuition-reliance, hasty judgment |
| Systems Thinking | Understand interactions and effects across the entire system | Analytical reasoning, long-term thinking | Fragmented approach, shortsightedness |
| Long-Term Thinking | Prioritize long-term outcomes and sustainability over short-term gains | Systems thinking, strategic planning | Short-term pressure, impulsive decisions |
| Critical Thinking | Objectively evaluate information and arguments; identify logical fallacies | Analytical reasoning, skepticism | Uncritical acceptance, authority-reliance |
| Creative Problem Solving | Generate novel solutions by breaking from conventional frameworks | Openness, abstract thinking | Rigid thinking, precedent-dependence |
| Risk Assessment | Identify and analyze potential risks; predict impacts | Analytical reasoning, caution | Haste, optimistic bias |
| Knowledge Structuring | Systematically organize and categorize scattered information | Systems thinking, pattern recognition | Random approach, surface-level analysis |
| Pattern Recognition | Discover recurring patterns and rules in data or situations | Analytical reasoning, data interpretation | Random approach, pattern blindness |
| Quantitative Reasoning | Analyze numerical data and reason through mathematical models | Evidence-based, statistical thinking | Intuition-only reliance, number avoidance |

### Behavioral Traits

| Trait | Definition | Synergy | Conflict |
|-------|-----------|---------|---------|
| Systematic Execution | Break work into steps per plan and execute procedurally | Analytical reasoning, thoroughness | Improvised approach, lack of planning |
| Evidence-Based Practice | Make decisions grounded in data and validated evidence | Analytical reasoning, quantitative reasoning | Intuition-reliance, speculation |
| Thoroughness | Pursue accuracy without missing even minor details | Systematic execution, evidence-based | Cutting corners, surface-level review |

### Dispositional Traits

| Trait | Definition | Synergy | Conflict |
|-------|-----------|---------|---------|
| Initiative | Proactively identify and solve problems without waiting for direction | Creative problem solving, pragmatism | Passive waiting, avoidance |
| Pragmatism | Pursue realistic and effective solutions over theoretical perfection | Initiative, evidence-based | Idealism, excessive perfectionism |
| Skepticism | Question the surface and actively search for hidden problems | Critical thinking, evidence-based | Blind acceptance, optimistic bias |
| User Empathy | Understand and consider the user's perspective and needs | Communication, adaptability | Technology bias, user neglect |
